# Instructions to Enable WordPress Debug Mode

## Via WP Engine File Manager:

1. Go to: https://my.wpengine.com/
2. Select: westgategroup
3. Click: "File Manager"
4. Open: wp-config.php
5. Find this line (near line 80):

   define('WP_DEBUG', false);

6. Replace with:

   define('WP_DEBUG', true);
   define('WP_DEBUG_LOG', true);
   define('WP_DEBUG_DISPLAY', false);
   define('SCRIPT_DEBUG', true);

7. Save the file

8. Try to access WordPress again

9. Via SFTP, download this file:
   /wp-content/debug.log

10. Send me the contents of debug.log

---

## What These Settings Do:

- WP_DEBUG = true → Enables error reporting
- WP_DEBUG_LOG = true → Writes errors to wp-content/debug.log
- WP_DEBUG_DISPLAY = false → Don't show errors on screen (security)
- SCRIPT_DEBUG = true → Use non-minified scripts (easier debugging)

---

## After Getting Errors:

Once you send me the error details, I'll:
1. Identify the exact file and line causing the issue
2. Fix it immediately
3. Deploy the fix
4. Turn debug mode back off
